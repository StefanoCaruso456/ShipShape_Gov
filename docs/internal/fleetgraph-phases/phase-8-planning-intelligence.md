# Phase 8: Planning Intelligence

Status: `future`

## What

Expand beyond execution drift into product and portfolio intelligence:

- capacity
- velocity
- scope creep
- roadmap generation
- burn-up / burn-down
- staffing pressure
- dependency and release confidence

## Why

This is where FleetGraph becomes more valuable to PMs, directors, and product owners, not just individual contributors.

## How

- add stronger planning primitives to Ship
- add historical planning state
- add roadmap and milestone structure
- reason on top of those signals with the same graph foundation

## Purpose

Give the LLM a stronger product foundation so it can reason about planning, prioritization, and delivery risk.

## Outcome

- planning-aware FleetGraph
- stronger PM / PO workflows
- better connection between roadmap and execution

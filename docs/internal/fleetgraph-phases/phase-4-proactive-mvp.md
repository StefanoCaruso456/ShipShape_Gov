# Phase 4: Proactive MVP

Status: `planned`

## What

Ship one proactive detection end to end:

- trigger
- detection
- routing
- surfaced output
- cooldown / snooze memory

## Why

This is the first assignment-critical product moment where FleetGraph acts without being asked.

## How

- use a hybrid trigger:
  - event-driven
  - 5-minute backstop sweep
- choose one narrow MVP use case
- surface the finding in a native Ship output

## Purpose

Prove that proactive mode is real, useful, and traceable.

## Outcome

- one real proactive flow on live Ship data
- one clean trace path
- one problem-detected trace path

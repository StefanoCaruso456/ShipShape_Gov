# Phase 5: On-Demand UI

Status: `planned`

## What

Add the embedded contextual FleetGraph chat surface.

## Why

The assignment requires on-demand mode, and the chat has to feel native to the user’s current Ship context.

## How

- embed FleetGraph inside an existing Ship context surface
- pass **Active View Context** into the graph
- support one narrow MVP question
- keep the answer grounded in fetched evidence

## Purpose

Make FleetGraph usable from the current page or tab without becoming a standalone chatbot.

## Outcome

- embedded contextual invocation
- same graph as proactive mode
- real page-aware on-demand behavior
